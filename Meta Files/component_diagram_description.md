## Internal Components

### Django MVC Backend

Основной компонент Django приложения, реализующий паттерн Model-View-Template (MVT). Обрабатывает HTTP запросы, отображает страницы, управляет пользовательским интерфейсом.

**Основные функции:**
- Обработка HTTP запросов через Views
- Рендеринг HTML страниц через Templates
- Управление пользовательскими сессиями
- Административная панель

**Взаимодействия:**
- Использует **Database** для хранения и получения данных
- Взаимодействует с **Internal API** для обработки AJAX запросов

### Internal API

REST API для взаимодействия фронтенда (веб-интерфейса) с бэкендом. Предоставляет JSON endpoints для веб-приложения.

**Основные функции:**
- Обработка AJAX запросов от фронтенда
- Управление лабораторными работами и экзаменами
- Интеграция с PNET и Elasticsearch
- Экспорт данных

**Основные endpoints:**
- `get_solutions()` - Получение решений
- `get_time()` - Получение оставшегося времени
- `get_pnet_auth()` - Авторизация PNET
- `create_pnet_lab_session()` - Создание сессии PNET
- `check_task_answers()` - Проверка ответов на задания
- `export_grades_xlsx()` - Экспорт оценок
- И другие endpoints для веб-интерфейса

**Взаимодействия:**
- Использует **Database** для доступа к данным
- Взаимодействует с **PNET Platform** для управления лабораторными работами
- Взаимодействует с **Elasticsearch/Kibana** для работы с логами

### Open API

Открытое API для взаимодействия с виртуальными машинами. Предоставляет endpoints, которые вызываются непосредственно с виртуальных машин лабораторных работ.

**Основные функции:**
- Получение заданий при запуске лабораторной работы
- Сохранение результатов выполнения заданий

**Основные endpoints:**
- `start_lab` - Вызывается при запуске лабораторной работы на ВМ
  - Получает информацию о заданиях
  - Возвращает конфигурацию для ВМ
- `end_lab` - Вызывается при завершении лабораторной работы на ВМ
  - Сохраняет результаты выполнения
  - Обновляет статус выполнения

**Особенности:**
- Endpoints доступны без аутентификации (используется `@csrf_exempt`)
- Принимают JSON данные от виртуальных машин
- Работают с базой данных для сохранения результатов

**Взаимодействия:**
- Получает запросы от **Virtual Machines**
- Использует **Database** для сохранения данных

### Scheduler

Планировщик задач на основе APScheduler для выполнения периодических задач по очистке завершенных лабораторных работ.

**Основные функции:**
- Автоматическое удаление завершенных лабораторных работ
- Очистка старых записей выполнения задач

**Cron Jobs:**
- **delete_labs_job** - Удаление завершенных лабораторных работ (каждый час)
- **delete_competition2team_job** - Удаление завершенных командных соревнований (каждые 15 минут)
- **delete_old_job_executions** - Очистка старых записей (каждый понедельник)

**Взаимодействия:**
- Использует **Database** для поиска завершенных работ
- Взаимодействует с **PNET Platform** для удаления лабораторных работ с платформы

## External Systems

### Database

База данных для хранения всех данных системы.

**Типы баз данных:**
- **SQLite** - Используется в режиме разработки
- **PostgreSQL** - Используется в продакшене

**Хранимые данные:**
- Модели лабораторных работ, экзаменов, соревнований
- Пользователи, команды, взводы
- Ответы на задания
- Конфигурации и метаданные

**Взаимодействия:**
- Используется всеми внутренними компонентами для хранения и получения данных

### PNET Platform

Платформа для виртуальных лабораторных работ. Предоставляет API для управления лабораторными работами, пользователями, топологией сетей.

**Основные функции:**
- Управление лабораторными работами
- Управление пользователями
- Работа с топологией сетей
- Управление виртуальными машинами
- Сессии лабораторных работ

**Взаимодействия:**
- Получает запросы от **Internal API** для управления лабораторными работами
- Получает запросы от **Scheduler** для удаления завершенных работ

### Elasticsearch/Kibana

Система логирования и визуализации данных.

**Основные функции:**
- **Elasticsearch** - Хранилище логов и данных
- **Kibana** - Интерфейс для визуализации и анализа данных

**Взаимодействия:**
- Получает запросы от **Internal API** для создания пользователей и работы с логами
- Используется для визуализации данных лабораторных работ

### Virtual Machines

Виртуальные машины лабораторных работ, на которых выполняются задания.

**Основные функции:**
- Выполнение заданий лабораторных работ
- Отправка результатов через Open API
- Предоставление SSH доступа для размещения флагов

**Взаимодействия:**
- Выполняет запросы к **Open API** для получения заданий и отправки результатов
- Управляется через **PNET Platform**

## Component Interactions Summary

### Request Flow: Web Frontend → Backend

1. **Web Browser** → **Django MVC Backend** (HTTP запросы для страниц)
2. **Django MVC Backend** → **Database** (чтение данных)
3. **Web Browser** → **Internal API** (AJAX запросы)
4. **Internal API** → **Database** (чтение/запись данных)
5. **Internal API** → **PNET Platform** (управление лабораторными работами)
6. **Internal API** → **Elasticsearch/Kibana** (работа с логами)
7. **Internal API** → **Web Browser** (JSON ответы)

### Request Flow: Virtual Machine → Backend

1. **Virtual Machines** → **Open API** (HTTP запросы start_lab, end_lab)
2. **Open API** → **Database** (сохранение результатов)
3. **Open API** → **Virtual Machines** (JSON ответы с заданиями)

### Scheduled Tasks Flow

1. **Scheduler** → **Database** (поиск завершенных лабораторных работ)
2. **Scheduler** → **PNET Platform** (удаление лабораторных работ с платформы)
3. **Scheduler** → **Database** (обновление статуса удаленных работ)

## Security Considerations

### Internal API
- Использует Django аутентификацию
- Требует авторизованных пользователей для большинства endpoints
- Использует CSRF защиту

### Open API
- Использует `@csrf_exempt` для доступа от виртуальных машин
- Валидирует данные перед сохранением
- Ограничивает доступ к определенным endpoints

### Database
- Использует параметризованные запросы для предотвращения SQL инъекций
- Поддерживает транзакции для целостности данных

### External Systems
- Использует HTTPS для связи с PNET и Elasticsearch
- Хранит учетные данные в безопасном виде
- Использует сессии и cookies для аутентификации
