# Package Diagram Description - Cyberpolygon Web Service

## Legend

### Architecture Layers

- **🔵 Data Management Layer** - Модели данных и миграции базы данных
- **🟢 Business Logic Layer** - Бизнес-логика, формы, интеграции
- **🟡 Application Layer** - Представления, API и фронтенд

### Arrows on the Diagram

Стрелки показывают зависимости между пакетами:
- Использование классов из других пакетов
- Вызовы методов
- Взаимодействие компонентов

## Data Management Layer

### Models

#### Лабораторные работы
- **Lab** - Модель лабораторной работы
  - Хранит метаданные (название, описание, slug)
  - Содержит JSON-данные топологии (ноды, коннекторы, сети)
  - Поддерживает платформы: PNET, CMD, без платформы
  
- **LabTask** - Задание лабораторной работы
  - Может содержать JSON-конфигурацию для автоматической проверки
  - Поддерживает текстовые вопросы и ответы
  
- **LabLevel** - Вариант лабораторной работы
  
- **LabNode** - SSH-нода для размещения флагов (логин, пароль)

#### Пользователи и команды
- **User** - Пользователь системы
  - Связан со взводом (Platoon)
  - Имеет учетные данные для PNET
  
- **Platoon** - Взвод (учебная группа)
  
- **Team** - Команда для соревнований

#### Экзамены и соревнования
- **Competition** - Экзамен (назначенная лабораторная работа)
  - Имеет временные рамки (start, finish)
  - Связана с лабораторной работой и заданиями
  
- **Competition2User** - Связь пользователя с экзаменом
  - Хранит сгенерированные флаги для заданий
  - Метаданные развертывания
  - Список проваленных заданий
  
- **TeamCompetition** - Соревнование (командное соревнование)
  
- **TeamCompetition2Team** - Связь команды с соревнованием
  
- **Answers** - Ответы пользователей/команд на задания

#### ККЗ (Контрольная курсовая работа)
- **Kkz** - Контрольная курсовая работа
  - Может иметь единые задания для всех участников
  
- **KkzLab** - Лабораторная работа в ККЗ
  
- **KkzPreview** - Превью случайного распределения заданий

#### Файлы
- **MyAttachment** - Прикрепленные файлы

### MigrationUtils

- **DatabaseAwareMigration** - Универсальные миграции для разных СУБД (SQLite/PostgreSQL)
  - Автоматически определяет тип базы данных
  - Применяет соответствующий SQL-код

## Business Logic Layer

### Forms

Формы для создания и редактирования экзаменов, соревнований, ККЗ и лабораторных работ. Обрабатывают валидацию данных и сохранение в базу данных.

#### Формы экзаменов
- **CompetitionForm** - Форма создания/редактирования экзамена
  - Создает записи Competition2User
  - Управляет распределением заданий
  
- **TeamCompetitionForm** - Форма создания соревнования
  
- **SimpleCompetitionForm** - Упрощенная форма создания экзамена

#### Формы ККЗ
- **KkzForm** - Форма создания ККЗ
  
- **SimpleKkzForm** - Упрощенная форма ККЗ
  
- **KkzLabInlineForm** - Инлайн форма для лабораторных работ в ККЗ

#### Прочие формы
- **LabForm** - Форма лабораторной работы
- **CustomUserCreationForm** - Форма создания пользователя
- **SignUpForm** - Форма регистрации
- **LabAnswerForm** - Форма ответа на задание
- **ChangePasswordForm** - Форма смены пароля

### PnetSessionManager

Управление сессиями PNET. Обеспечивает создание и удаление лабораторных работ, управление пользователями, работу с топологией. Использует EveFunctions для взаимодействия с PNET API.

### LabTopology

Работа с топологией лабораторных работ. Парсинг JSON-данных топологии, получение списка SSH-нод, поиск нод по имени.

### SSHManager

SSH-подключения к нодам лабораторных работ. Размещение флагов на нодах, выполнение команд через SSH. Использует библиотеку Fabric для управления соединениями.

### FlagGenerator

Генерация уникальных флагов для заданий. Создает флаги длиной 8-12 символов из букв и цифр.

### FlagDeployment

Развертывание флагов на нодах лабораторных работ через SSH. Асинхронная обработка через очередь задач. Использует FlagGenerator, SSHManager и LabTopology.

### EveFunctions

Функции работы с PNET API:
- Аутентификация: `pf_login`, `logout`
- Управление пользователями: `create_user`, `create_directory`, `change_user_password`
- Управление лабораторными работами: `create_lab`, `delete_lab`
- Работа с топологией: `get_lab_topology`
- Управление нодами: `turn_on_node`, `get_node_status`
- Сессии: `create_pnet_lab_session_common`, `get_session_id`, `leave_session`

### ElasticUtils

Работа с Elasticsearch и Kibana:
- Создание и удаление пользователей Elasticsearch
- Интеграция с Kibana для визуализации данных

### Config

Управление динамическими конфигурациями и фиче-флагами. Предоставляет функции для получения конфигурационных параметров с кэшированием:
- `get_pnet_url()` - URL сервера PNET
- `get_pnet_base_dir()` - Базовый путь к лабораторным работам в PNET
- `get_web_url()` - URL веб-сервера для внутренних запросов
- `get_student_workspace()` - Путь к рабочему пространству студента
- `cache_for_minutes()` - Декоратор для кэширования результатов функций

Использует `dynamic_config.utils.get_config` для получения значений из динамической конфигурации. Используется всеми модулями бизнес-логики и приложения для доступа к настройкам системы.

## Application Layer

### Views

Django представления (класс-базированные views и функции) для отображения страниц и обработки запросов.

#### Представления лабораторных работ
- **LabDetailView** - Детальная страница лабораторной работы
- **LabListView** - Список лабораторных работ

#### Представления экзаменов и соревнований
- **CompetitionDetailView** - Детальная страница экзамена
- **CompetitionListView** - Список экзаменов
- **CompetitionHistoryListView** - История экзаменов
- **TeamCompetitionDetailView** - Детальная страница соревнования
- **TeamCompetitionListView** - Список соревнований

#### Представления ККЗ
- **KkzDetailView** - Детальная страница ККЗ
- **CreateKkzView** - Создание ККЗ
- **CreateKkzFromLabView** - Создание ККЗ из лабораторной работы

#### Представления пользователей
- **UserDetailView** - Профиль пользователя
- **PlatoonDetailView** - Детальная страница взвода
- **PlatoonListView** - Список взводов

#### API Views
- **AnswerAPIView** - REST API для ответов (CRUD операции)

#### Утилиты
- **UtilsConsole** - Консоль утилит
- **KibanaDashboard** - Дашборд Kibana
- **CustomSummernoteUploadAttachment** - Загрузка вложений

### API

REST API endpoints для взаимодействия фронтенда с бэкендом. Обрабатывают запросы к PNET, Elasticsearch, генерацию отчетов.

#### API лабораторных работ
- `start_lab()` - Запуск лабораторной работы
- `end_lab()` - Завершение лабораторной работы
- `get_pnet_auth()` - Получение авторизации PNET
- `create_pnet_lab_session()` - Создание сессии PNET
- `check_availability()` - Проверка доступности

#### API экзаменов и соревнований
- `get_solutions()` - Получение решений
- `get_time()` - Получение оставшегося времени
- `press_button()` - Обработка действий (продление времени, удаление)
- `check_updates()` - Проверка обновлений

#### API заданий
- `check_task_answers()` - Проверка ответов на задания
- `get_user_tasks_status()` - Статус заданий пользователя
- `load_tasks()` - Загрузка заданий
- `load_levels()` - Загрузка вариантов

#### API ККЗ
- `kkz_preview_random()` - Превью случайного распределения заданий
- `kkz_save_preview()` - Сохранение превью

#### API экспорта
- `export_grades_xlsx()` - Экспорт оценок в Excel

#### API Kibana
- `get_kibana_auth()` - Авторизация Kibana
- `check_kibana_auth_status()` - Проверка статуса авторизации

### FrontendComponents

JavaScript контроллеры для взаимодействия с API и управления пользовательским интерфейсом.

#### Контроллеры платформ
- **PnetController** - Контроллер для PNET платформы
  - Аутентификация в PNET
  - Создание сессии лабораторной работы
  - Перенаправление на топологию
  
- **CmdController** - Контроллер для CMD режима
  - Создание консольной сессии
  
- **PnetIframeController** - Контроллер iframe для PNET

#### Контроллеры данных
- **SolutionsController** - Контроллер решений
  - Загрузка и обновление решений
  
- **TasksController** - Контроллер заданий
  - Загрузка заданий
  - Отправка ответов
  
- **SolutionsTable** - Таблица решений

#### Контроллеры UI
- **TimerController** - Контроллер таймера
  - Запуск и обновление таймера
  
- **ControlButtons** - Контроллер кнопок управления
  - Обработка нажатий кнопок
  
- **NavbarController** - Контроллер навигации
- **LabDetail** - Детали лабораторной работы

#### Интеграции
- **KibanaController** - Контроллер Kibana
  - Аутентификация в Kibana
  - Загрузка дашборда
  
- **ExportRating** - Экспорт рейтинга
  - Экспорт в Excel

#### Утилиты
- **ConsoleLoader** - Загрузчик консоли
  - Отображение загрузчика
  - Обновление текста загрузки
  
- **CommonUtils** - Общие утилиты
  - Выполнение API запросов

## Dependencies Between Layers

### Data Management Layer → Business Logic Layer
- **Models** используются всеми компонентами бизнес-логики для доступа к данным
- **MigrationUtils** обеспечивает миграции моделей для разных СУБД

### Business Logic Layer → Application Layer
- **Forms** используются в Views для обработки пользовательского ввода
- **API** предоставляет REST endpoints для фронтенд контроллеров
- **PnetSessionManager**, **FlagDeployment**, **ElasticUtils** используются через API и Forms

### Взаимодействия внутри Business Logic Layer
- **Forms** → **PnetSessionManager** (создание лабораторных работ в PNET)
- **Forms** → **FlagDeployment** (развертывание флагов при создании экзаменов/соревнований)
- **Forms** → **ElasticUtils** (создание пользователей Elasticsearch)
- **Forms** → **Config** (получение конфигурационных параметров)
- **PnetSessionManager** → **EveFunctions** (вызовы PNET API)
- **PnetSessionManager** → **Config** (получение URL и путей PNET)
- **FlagDeployment** → **FlagGenerator** (генерация флагов)
- **FlagDeployment** → **SSHManager** (размещение флагов на нодах)
- **FlagDeployment** → **LabTopology** (получение информации о нодах)
- **FlagDeployment** → **EveFunctions** (работа с PNET сессиями)
- **FlagDeployment** → **Config** (получение URL PNET)
- **EveFunctions** → **Config** (получение конфигурационных параметров)

### Взаимодействия внутри Application Layer
- **Views** → **Models** (чтение данных из базы)
- **Views** → **Forms** (обработка форм)
- **Views** → **PnetSessionManager** (управление сессиями)
- **Views** → **Config** (получение конфигурационных параметров)
- **API** → **Models** (доступ к данным)
- **API** → **PnetSessionManager** (управление сессиями)
- **API** → **LabTopology** (работа с топологией)
- **API** → **EveFunctions** (вызовы PNET API)
- **API** → **Config** (получение конфигурационных параметров)
- **FrontendComponents** → **API** (вызовы REST endpoints)

### Config как общий компонент
- **Config** используется всеми модулями Business Logic Layer и Application Layer для получения конфигурационных параметров
- **Models** → **Config** (получение путей и URL при сохранении)
